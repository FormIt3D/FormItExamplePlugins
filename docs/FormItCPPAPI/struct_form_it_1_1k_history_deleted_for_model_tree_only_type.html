FormIt::kHistoryDeletedForModelTreeOnlyType Struct Reference

Description

This is a hack to make sure the model tree is updated correctly since we cannot use kModelChanged as WSM::ModelChangeData::Type::DeletedHistory is not broadcast (instead DeleteHistoryCaches is used).

Either we need an API in core to expose the History cache mechanism or we need to broadcast the DeletedHistory message. Payload: History ID
