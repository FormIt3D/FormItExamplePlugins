FORMIT_CORE_EXPORT bool FormIt::Memory::IsMemoryLow ( size_t  checkInterval = 100,
double  minMemory = 0.05 
)

Returns true if memory is running low and operations that are in the process of allocating more memory should abort.

checkInterval is used to avoid a possible costly calculation of free memory. If set to a value larger than 1, the free memory is calculated only when this method is invoked the checkInterval-th time in a row. Typically this method is called in a loop that continues to allocate bits of memory. Set the checkInterval as high a possible. A good rule of thumb would be loopCount / 1000. That means that the memory is recalculated at the most 1000 times during the execution of the loop. If checkInterval is passed as 0, the internal counter is reset so the memory will be checked again This is good practice at the end of the loop and after memory became low and the loop was aborted or allocated memory was freed minMemory is the threshold of what low memory means. 5% free by default
