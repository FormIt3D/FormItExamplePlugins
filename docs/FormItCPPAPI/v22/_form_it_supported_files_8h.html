Why Using Macro ?!

Most of the time the rule is : "Avoid as much as possible!" However times to times it provides some benefits... so here, I'm using macros for the following reasons:

  1. It Force naming convention for classes declaration (see Examples)
  2. By forcing some methods declaration (like class Constructor), it allows some code factoring
  3. It allows me to avoid to re-write too much thing in formIT current code AND still give me the possibility to extends preexisting classes with Import

Note: I keep here the sames class and names as before for xxxExportOptions classes. This solution can be taken as an interim solution before a potential larger code refactoring.

Macro usage examples:

Example [1] Creates classes for AXM format:

DECL(Axm, FileTypes::AXM, "FormIt AXM file(*.axm)", "axm") -> declare 'struct AxmBaseOptions : FileOptions'
DECL_IMPORT(Axm) -> declare 'struct AxmImportOptions : AxmBaseOptions' - without any import options
DECL_EXPORT(Axm) -> declare 'struct AxmExportOptions : AxmBaseOptions' - without any import options
